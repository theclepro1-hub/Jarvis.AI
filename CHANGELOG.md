# История изменений

## [20.1.5] - 2026-04-01

- стабилизирована левая колонка `Control Center`: ширина бокового меню больше не прыгает между разделами, а карточка `Нубика` стала компактнее и перестала съедать нижний текст
- control center перестал конфликтовать со стартовым activation gate: при открытии настроек первый overlay теперь снимается, а при закрытии возвращается только если активация ещё не завершена
- маршрутизация секций настроек собрана в явные section-modules `main/apps/voice/diagnostics/updates/system`, а системная вкладка больше не внедряется в `SettingsUiMixin` через runtime monkey-patch
- сохранение главных настроек переведено на единый non-modal lifecycle: предупреждения и итог save больше не дёргают `messagebox` на каждом шаге, используется inline status/toast
- scroll-контур ушёл с глобального `bind_all` на локальные top-level bindings, что делает длинные страницы настроек предсказуемее на больших экранах
- аудиоустройства получили устойчивые `device signature`: выбор микрофона и вывода сначала восстанавливается по подписи, а не по плавающему индексу
- добавлен живой smoke на конфликт `startup gate <-> settings`, а `pytest`, `ruff`, `compileall`, `unit_checks` и `crash_test` остаются зелёными на этой версии

## [20.1.2] - 2026-04-01

- переписан фундамент масштабирования: теперь сохраняется системный baseline `tk scaling`, пользовательский диапазон `90..150%` применяется честно, а рабочий shell использует pixel-fonts без двойного масштабирования
- центр управления перестал жить как stale overlay: закрытие уничтожает окно полностью, а после UI-save оно пересобирается заново, потому что Tk не гарантирует корректную динамическую подстройку уже созданных виджетов после смены `tk scaling`
- устранён слом скролла после сохранения: пересборка workspace больше не теряет `control center` как scroll target, а smoke/crash-проверки обновлены под новый жизненный цикл окна настроек
- activation gate первого запуска теперь поднимается сразу после построения UI через `after_idle`, без флакного таймера `after(180)`

## [20.1.1] - 2026-04-01

- убран релизный blocker: в `jarvis_ai/runtime_voice.py` удалён дублирующийся `play_last_voice_capture`, из-за которого `ruff` ронял release workflow
- центр управления перестал сам себя дёргать: scroll/layout refresh теперь коалесцируется, повторные `show_section`/`update_idletasks()`-цепочки при открытии убраны, а resize во время открытых настроек больше не вызывает тяжёлую перестройку на каждый тик
- сохранение аудионастроек стало честнее: после смены микрофона голосовой контур принудительно переоткрывает устройство, а восстановление выбранного микрофона теперь умеет находить его по имени, если индекс в системе сдвинулся
- подсвечены основные кликабельные действия в центре управления: ключевые кнопки получили явный `hand2`, чтобы меньше путаться на длинных страницах настроек
- локальный `build_release.ps1` выровнен с CI: перед сборкой теперь обязательно гоняются `ruff`, `pytest` и полный `compileall`, а PyInstaller запускается через тот же Python, что и остальные quality gates
- тексты rollback/update сделаны честнее: откат из UI гарантирует профиль и пользовательские данные, но не обещает автоматический возврат установленного `.exe` там, где это фактически невозможно
- обновлены релизные метаданные и версия продукта до `20.1.1`

## [20.0.0] - 2026-03-31

- отключён `ui_rewrite` как активный runtime: приложение снова собирается на стабильном `shell/control-center` фундаменте без конфликтующего второго UI-слоя
- возвращён красивый chat-first shell: левая панель разделов, asset-кнопки `send` и `mic`, кликабельный Нубик и верхние status-chip’ы снова стали основным маршрутом
- убрана правая helper-панель из чата: справа больше не живёт второй Нубик, центр снова принадлежит разговору
- восстановлена встроенная регистрация первого запуска: activation gate снова отдельный полноэкранный слой внутри того же окна, до активации чат и его кнопки блокируются
- полноэкранный центр настроек снова использует общий стабильный layout и единую навигацию вместо набора canvas-страниц из `ui_rewrite`
- smoke- и crash-проверки переписаны под реальный shell, а не под мёртвый объект `app.ui_rewrite`

## [19.1.1] - 2026-03-31

- исправлен новый критический `.exe`-краш `RecursionError: maximum recursion depth exceeded`: в новом screen runtime убран рекурсивный цикл между `JarvisApp.refresh_workspace_layout_mode()` и `ui_rewrite.refresh_layout()`
- исправлен критический `.exe`-краш: добавлен runtime-слой совместимости для `groq` и `httpx>=0.28`, из-за которого приложение падало на старте с ошибкой `Client.__init__() got an unexpected keyword argument 'proxies'`
- вынесены `bootstrap`, `system_actions`, `response_parsing`, `structured_logging` и единый `action_catalog`, а `JarvisApp` получил явные контроллеры `UiShell`, `ConversationController`, `VoiceController`, `WindowController` и `ActionExecutor`
- стабилизирован первый запуск и shell: убрано двойное расписание activation gate, заменены проблемные label-кнопки на реальные `Button`, зафиксирована геометрия `Нубика`, уменьшены лишние перестроения layout
- добавлены guardrails релиза: `ruff`, smoke-test безопасного старта, тест на консистентность релизной версии и pre-build проверка метаданных

## [18.1.1] - 2026-03-31

## [18.0.0] - 2026-03-29

- начат аварийный цикл стабилизации shell под реальные жалобы: сильное дёрганье, пустые зоны, неудачный narrow-layout и перегруженный центр настроек
- на главном экране убрана отдельная `chat-side` панель, чтобы разговор занимал больше полезной ширины
- voice meter перенесён в верхнюю рабочую зону рядом с header, а для узких ширин добавлена компактная секционная навигация
- добавлен отдельный пакет `jarvis_ai/ui_actions/` как база для централизованной маршрутизации UI-действий
- переписан `TASKS.md` в подробный backlog релиза `18.0.0` с аварийным блоком, требованиями со скринов, архитектурой, тестами и релизными задачами
- улучшено копирование истории чата, усилена вставка в composer и закрыт явный runtime-баг `APP_CLOSE_EXES is not defined`
- начата укрупнённая русификация и типографическая чистка `Центра управления`
- регистрация и базовая настройка встроены в тот же UI-контур, что и чат с центром управления, без обязательного отдельного мастера
- добавлены `?`-пояснения для ключевых настроек, обновлений и быстрых полей, а guide-panel теперь использует `assets/noob2.png`, если он присутствует
- добавлен пакет `audio_runtime`: убран прямой `audioop` из кода приложения, централизованы проверка `ffmpeg` и suppress-path для `pydub`
- добавлен разбор сценариев прямо из системного экрана: можно посмотреть, почему сценарий сработал или не сработал
- `Диагностика ZIP` переименована в нормальный `Пакет поддержки`, а корневые probe/tmp-скриншоты исключены из Git-инвентаря через `.gitignore`
- тестовый контур расширен до `16 passed`, а `scripts/crash_test.py` проходит `42/42`

## [17.5.0] - 2026-03-29

- полностью переписан runtime shell: новый режимный layout `wide / compact / minimal / focus` заменил слой аварийных патчей поверх старого домашнего экрана
- убраны дублирующие верхние кнопки на широком экране; в header осталась только `Меню`, а навигация живёт в sidebar и в popup для узкого режима

- исправлены строки выбора опасных действий в `SetupWizard` и добавлен такой же выбор во встроенную активацию первого запуска
- успокоен wrap-helper для overlay-экранов, чтобы текст и иконки меньше «летали» на первом открытии настроек, регистрации и fullscreen

- fullscreen-переключение теперь проходит через короткий скрытый переход вместо видимого «плывущего» пересчёта

- верхний live-context и нижний composer уплотнены, чтобы чат получил больше полезной высоты на главном экране
