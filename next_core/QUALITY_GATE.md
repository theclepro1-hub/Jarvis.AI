# QUALITY GATE

Ни одна версия JARVIS NEXT не считается готовой, пока не проходит все пункты ниже.

## UI-качество

1. `0` клипнутых контролов на `1366x768`.
2. `0` клипнутых контролов на `1600x900`.
3. `0` клипнутых контролов на `1920x1080`.
4. `0` дерганий layout при 50 resize-циклах.
5. `0` скачков между wide и compact без реального пересечения порога.
6. Кнопка `Меню` отсутствует в wide/fullscreen и появляется только в compact.
7. В wide-режиме чат занимает не меньше `68%` рабочей ширины shell.
8. В compact-режиме исчезают тяжёлые боковые панели, а не чат.

## UX-качество

1. Пользователь понимает, где он находится, за первые 10 секунд.
2. Первый запуск не требует догадок.
3. Каждая важная настройка имеет короткое пояснение через `?`.
4. Нет мёртвых кнопок.
5. Нет пустых декоративных блоков без смысла.
6. Нет дублирующих экранов для одного и того же сценария.

## Текст и локализация

1. `0` битых строк.
2. `0` mojibake.
3. `0` англоязычных хвостов в пользовательском слое без причины.
4. Текст объясняет смысл функции, а не повторяет её название.

## Поведение

1. First-run проходит без отдельного legacy-popup сценария.
2. Вставка работает везде: `Ctrl+V`, `Shift+Insert`, мышь.
3. Копирование из чата работает.
4. Настройки реально сохраняются и применяются.
5. Dry-run показывается до опасного действия.
6. После крэша доступен recovery-flow.

## Voice

1. Voice meter не ломает UI и не душит поток.
2. Пользователь видит живой статус микрофона.
3. STT/TTS ошибки показываются человечески.
4. Нет ощущения “приложение зависло”, если внешний провайдер не ответил.

## Product gate

Каждая новая фича проходит вопрос:

1. Это точно должно жить в главном shell?
2. Это можно спрятать в drawer или режим?
3. Это помогает основной задаче или просто раздувает продукт?

Если ответ слабый, фича не идёт в релиз.

## Release gate

Релиз запрещён, если не пройдены:

- unit checks
- UI smoke
- resize stress
- first-run acceptance
- settings acceptance
- compact/wide acceptance
- build smoke

## Definition of done

Фича считается готовой только если:

1. реализована;
2. визуально чиста;
3. не ломает shell;
4. прошла сценарии использования;
5. не увеличила хаос продукта.
