# ROADMAP

## Фаза 0. Заморозка legacy

Цель:

- считать старый `jarvisAI 2.0` legacy-базой;
- не тянуть его UI-хаос в новый продукт;
- забирать из него только полезную логику.

## Фаза 1. Основа

Сделать:

- новый shell;
- новый design system;
- пролог;
- first-run;
- главный composer;
- compact/wide layout;
- quality harness для UI.

Результат:

появляется пустой, но цельный JARVIS NEXT.

## Фаза 2. Исполнение

Сделать:

- action pipeline;
- dry-run;
- recovery;
- history;
- context drawer;
- system execution layer.

Результат:

JARVIS не просто выглядит как продукт, а уже реально выполняет задачи.

## Фаза 3. Roblox

Сделать:

- Roblox mode;
- Luau assistance;
- asset-first recommendations;
- project context;
- gameplay/system templates.

Результат:

JARVIS становится сильным dev-assistant для Roblox.

## Фаза 4. Recreate / Create

Сделать:

- recreate mode;
- scene/interface reconstruction flows;
- creation workflows;
- better structured prompting and outputs.

Результат:

ядро начинает работать как универсальный production assistant.

## Правило roadmap

Новая функция входит в работу только если:

1. не ломает shell;
2. вписывается в манифест;
3. проходит quality gate;
4. не создаёт ещё один дублирующий сценарий.
