# JARVIS Unity 22.3.0

22.3.0 — это не декоративный апдейт, а жёсткий проход по контуру ассистента: голос, planner, Telegram, updater и системные действия Windows.

## Главное

- Wake / voice pipeline стал честнее:
  - отдельный handoff после слова активации;
  - лучшее подхватывание начала команды;
  - меньше ложного ощущения, что JARVIS уже всё понял, когда он только начал захват.
- Голосовые multi-action команды больше не завязаны только на запятые:
  - добавлен voice post-processor;
  - `открой ютуб музыку и прибавь` теперь нормализуется до отдельных действий заметно лучше.
- Planner стал строже к частичному распознаванию:
  - меньше ложной уверенности;
  - при неоднозначности JARVIS уточняет, а не делает вид, что понял всё идеально.
- Telegram transport стал устойчивее:
  - меньше дублей и "догоняющих" ответов;
  - выше пропускная способность очереди сообщений.
- AI modes получили реальные latency budgets:
  - `Быстро`, `Авто`, `Качество` теперь заметнее отличаются по поведению;
  - fallback и timeout стали агрессивнее в пользу живого ответа.
- Обновления доведены до законченного сценария Windows:
  - проверка релиза;
  - выбор installer asset;
  - скачивание и запуск установщика обновления.
- Windows integration добита сильнее:
  - AppUserModelID;
  - installer metadata;
  - нормальная идентичность приложения для ярлыков и панели задач.

## Системные команды

JARVIS лучше понимает встроенные Windows-действия из коробки:

- `открой параметры`
- `открой проводник`
- `открой диспетчер задач`
- `открой панель управления`
- `открой загрузки / документы / рабочий стол`
- `заблокируй экран`
- `выключи компьютер`
- `перезагрузи компьютер`
- `сон / гибернация / выход из системы`

## UX и устойчивость

- Защита от повторной отправки одинакового сообщения в чате.
- Busy/status-поведение для долгих AI-операций стало честнее.
- Автоскролл в чате больше не должен дёргаться, если пользователь читает историю.
- Generic `музыка` больше не должна молча улетать в системный Windows Media fallback.

## Для пользователей старого JARVIS

Эта версия ближе к обещанию "управлять ПК как набором скриптов, но в нормальном приложении", а не просто красиво отвечать в чате.

Ключевые отличия от ранних сборок:

- лучшее управление Windows из коробки;
- более честный voice pipeline;
- менее игрушечный planner;
- более живой Telegram-контур;
- реальный update/apply-сценарий вместо пустого checker-only UX.
