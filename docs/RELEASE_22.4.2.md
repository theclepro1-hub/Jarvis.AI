# JARVIS Unity 22.4.2

22.4.2 закрывает два реальных пользовательских дефекта в desktop-контуре: неполное распознавание системных power-команд и повторное проигрывание старого Telegram backlog после старта.

## Что вошло

- Добит локальный роутинг системных команд:
  - добавлены естественные формы для `restart`, `shutdown`, `sleep`, `logoff`, `lock`;
  - `перезапусти пк`, `выключение пк`, `сон`, `выйди из учетки`, `блокируй экран` и соседние варианты теперь должны уходить в локальный system-path, а не мимо него.
- Поставлен guard до AI fallback:
  - если фраза похожа на системную команду, но локально не совпала, приложение больше не должно придумывать советы вроде PowerShell-команд;
  - вместо этого возвращается локальное уточнение.
- Закрыт replay старых Telegram update при пустом offset:
  - при первом старте без `telegram_state.json` сервис теперь поднимает offset до актуального хвоста;
  - старый backlog не должен разъезжаться в повторный спам.
- Wake/voice runtime не менялся:
  - `core/voice` и wake backend в этом релизе не переписывались;
  - правки ограничены routing/telegram/release-контуром.

## Что проверено

- `pytest -q`
- `ruff check app core tests tools ui`
- `python -m compileall app core ui tests`
- таргетные regression-тесты на `intent_router`, `command_router`, `telegram_service`
- read-only smoke по voice/wake unit-тестам без изменения voice-кода

## Итог

22.4.2 — это точечный релиз на честную локальную маршрутизацию системных команд и на защиту Telegram от повторного старого backlog, без лишнего риска для уже стабилизированного wake/voice-контура.
