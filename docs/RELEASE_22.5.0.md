# JARVIS Unity 22.5.0

22.5.0 собирает пользовательские и технические правки в один чистый релиз: с более простым UI, живым wake-feedback, честным `assistant_mode` для voice/STT и без лишних лагов в настройках.

## Что изменилось

- Почищен основной UI:
  - регистрация остаётся короткой и понятной;
  - выбор AI-режима остаётся в конце onboarding;
  - настройки по умолчанию свёрнуты;
  - шапка снова показывает живой статус ассистента вместо глухого `Готов`;
  - секции настроек и advanced-блок стали компактнее и чище.
- Нубик снова работает как проводник:
  - подсказки читаемые;
  - помощь подхватывается по hover и клику на секциях и настройках.
- Wake и voice-path приведены к более честному поведению:
  - wake-handoff снова видно внизу чата как короткий transient-статус;
  - служебные фразы после wake не подменяют пользовательскую команду перед отправкой в чат;
  - `voice` и `stt` теперь подчиняются единому `assistant_mode`, а не живут отдельно на старом `voice_mode`.
- Голосовой UI и настройки стали быстрее:
  - локальная диагностика Llama больше не стопорит экран настроек синхронным probe;
  - lazy-сервисы создаются под блокировкой и не дают редких гонок на фоне;
  - экран проверки голоса снова показывает, что было услышано и что будет сделано.
- Локальный AI и advanced-настройки ведут себя честно:
  - `private` не притворяется готовым без локальной модели;
  - `auto` не врёт о готовности `llama.cpp` или `Ollama`;
  - статус локальной модели зависит от реального runtime на машине;
  - для обычного пользователя служебные статусы снова краткие и понятные.
- Приватный режим теперь можно подготовить без giant installer:
  - приложение сначала пытается поднять portable local runtime через официальный Windows zip от Ollama;
  - runtime и локальные модели складываются в `%LOCALAPPDATA%\\JarvisAi_Unity\\runtime`;
  - если portable-путь не поднялся, приложение открывает официальный `OllamaSetup.exe` как fallback;
  - обычному пользователю это показывается как одна кнопка `Подготовить локальный режим`, а не как набор backend-ручек.

## Локальная Llama

Локальный runtime считается реально готовым только если выполнено одно из условий:

- для `llama.cpp`:
  - установлен `llama-cpp-python`
  - указан валидный путь к `.gguf`
- для `Ollama`:
  - локальный daemon отвечает
  - нужная модель действительно установлена

Если этого нет, интерфейс и backend теперь должны говорить об этом прямо и не выдавать локальный режим за готовый.
