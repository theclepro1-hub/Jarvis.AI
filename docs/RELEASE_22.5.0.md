# JARVIS Unity 22.5.0

22.5.0 собирает пользовательские и технические правки в один чистый релиз: без битой кириллицы, без debug-мусора в главных экранах, с честным поведением локального AI и с более аккуратным voice/wake UX.

## Что изменилось

- Почищен основной UI:
  - регистрация снова короткая и понятная;
  - выбор AI-режима остаётся в конце onboarding;
  - пропуск первого запуска убран;
  - настройки по умолчанию свёрнуты;
  - debug, route-диагностика и лишняя техничка убраны из пользовательских экранов.
- Нубик снова работает как проводник:
  - подсказки нормальные и читаемые;
  - помощь подхватывается не только по боковым вкладкам, но и по секциям настроек.
- Wake и voice-path приведены к более честному поведению:
  - wake-handoff снова видно внизу чата как короткий transient-статус;
  - строгая wake-грамматика снова не срабатывает на короткие шумные алиасы;
  - служебные фразы после wake не подменяют пользовательскую команду перед отправкой в чат.
- Голосовой UI и быстрые экраны стали легче:
  - убраны лишние hover/tap-обработчики, которые давали лишний шум и лаги;
  - wake runtime больше не дёргает тяжёлый stop/restart прямо из обычных UI-переключателей;
  - экран приложений меньше пересчитывает каталоги и счётчики на каждый клик;
  - экран проверки голоса снова показывает, что было услышано и что будет сделано;
  - onboarding и режим ассистента стали компактнее, без лишних плашек.
- Локальный AI и advanced-настройки ведут себя честно:
  - `private` не притворяется готовым без локальной модели;
  - `auto` не врёт о готовности `llama.cpp` или `Ollama`;
  - статус локальной модели зависит от реального runtime на машине;
  - для обычного пользователя сверху больше не течёт служебный статус `слушаю/захватываю`.

## Локальная Llama

Локальный runtime считается реально готовым только если выполнено одно из условий:

- для `llama.cpp`:
  - установлен `llama-cpp-python`
  - указан валидный путь к `.gguf`
- для `Ollama`:
  - локальный daemon отвечает
  - нужная модель действительно установлена

Если этого нет, интерфейс и backend теперь должны говорить об этом прямо и не выдавать локальный режим за готовый.
