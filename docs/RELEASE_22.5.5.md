# JARVIS Unity 22.5.5

22.5.5 доводит релиз до пользовательского состояния: без битой кириллицы, без debug-мусора в интерфейсе и с честным поведением локальных AI-режимов.

## Что изменилось

- Почищен основной UI:
  - регистрация снова короткая и понятная;
  - выбор AI-режима перенесён в конец onboarding;
  - кнопка пропуска убрана;
  - настройки по умолчанию свернуты;
  - debug, route-диагностика и технический мусор убраны из пользовательских экранов.
- Возвращена нормальная работа Нубика:
  - подсказки снова читаемые;
  - помощь показывается не только по боковым вкладкам, но и по содержательным плашкам и секциям.
- Упрощены и вычищены экраны `Чат`, `Голос`, `Приложения`, `Настройки`.
- Режимы ассистента оставлены как главный пользовательский выбор:
  - `Быстрый`
  - `Стандартный`
  - `Умный`
  - `Приватный`
- Локальный AI теперь ведёт себя честно:
  - `private` не маскируется под готовый, если нет локальной модели;
  - `auto` не врёт о готовности `llama.cpp` или `Ollama`;
  - статус прямо зависит от реального `.gguf`, `llama-cpp-python`, локального Ollama daemon и установленной модели.
- Убран скрытый skip-сценарий регистрации: старый флаг пропуска больше не обходит onboarding молча.

## Важно про локальную Llama

Локальный текстовый runtime считается реально готовым только если выполнено одно из условий:

- для `llama.cpp`:
  - установлен `llama-cpp-python`
  - указан валидный путь к `.gguf`
- для `Ollama`:
  - локальный daemon отвечает на `http://localhost:11434/v1`
  - нужная модель действительно установлена

Если этого нет, интерфейс и backend теперь говорят об этом прямо и не выдают локальный режим за готовый.
